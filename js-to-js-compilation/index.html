This is a generic workflow for a compiler. Input source code is parsed into a token stream, or list, and then analyized into an abstract syntax tree, and the AST is transformed into the compiled output. In GCC c++ code is transformed into machine code, the CoffeeScript compiler transforms the coffeescript syntax into JavaScript. Babel accepts syntax that is often new and unsupported in current browsers and transforms that into code that *can* be run, with the same semantics.
var a = 42;
Babel is implemented using transforms. Your source code is read, parsed into a stream of tokens, a simple list. Then the list of tokens is analyzed into an abstract syntax tree. And finally transformed into the generated code.

Code transforms

If you've been paying attention to the Go community recently you'll know there was a big push by the Go team to rewrite the compiler. The team decided it would be a bad decision to do this by hand. So they wrote a compiler to transform all of the C code in the original Go compiler to being implemented in the Go language itself. So Go now uses a self-hosted compiler that was mostly rewritten automatically with a few pieces needing to be updated by hand.
